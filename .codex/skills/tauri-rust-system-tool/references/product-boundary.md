# 产品边界

## 意图

保持系统小工具的首版边界清楚，不把截图工具做成编辑器、资料库或内容容器。

## 适用场景

- 修改 `apps/capture` 的产品行为。
- 判断新功能是否进入 v1。
- 设计托盘、通知、缓存、设置项。

## 必须遵守的规则

- 默认无主窗口，常驻托盘。
- 区域截图后复制图片到剪贴板，写缓存文件，发系统通知。
- 点击通知打开系统默认图片查看器。
- 取消截图不改剪贴板、不写缓存、不发通知。
- Windows 是 v1 正式验收平台。

## 推荐模式

- 托盘菜单提供区域截图、当前屏幕截图、打开最近截图、设置、退出。
- 默认不弹保存框。
- 默认不进入应用内历史页。
- 缓存默认保留最近 50 张。

## 反模式

- v1 加标注、OCR、录屏、历史资料库。
- 点击通知回到应用内页面。
- 为跨平台预留牺牲 Windows 主链路清晰度。

## 证据

- `apps/capture/docs/product.md` 定义 v1 主流程、默认行为和不做范围。
- `PLAN.md` 记录 capture 是独立截图工具，不是 Arbor 容器页面。
