# KeyDock 产品说明

## 一句话

KeyDock 是一个 Windows 原生屏幕键盘。它像一个可停靠的系统工具面板，点击虚拟按键后向当前前台窗口输入，不抢走输入目标的焦点。

## 命名

产品名：`KeyDock`

含义：

- `Key` 表示键盘和按键输入
- `Dock` 表示停靠、常驻、贴近系统工具的面板形态
- 名字短、克制，不和 Windows 内置“屏幕键盘”“触摸键盘”混淆

## 产品定位

KeyDock 是通用桌面屏幕键盘，不是输入法、宏工具或自动化平台。

当前目标用户就是自己。重点场景是：

- 没有实体键盘时做基础输入
- 触控屏或鼠标场景下临时输入
- 在不打断目标窗口焦点的前提下补充键盘输入
- 后续作为 Win32/Rust 桌面小工具的边界样板

## 产品气质

- 系统工具感
- 低噪声视觉
- 明确状态反馈
- 不做拟物键盘
- 不做花哨动效
- 默认设置就应该能用

## v1 主流程

1. 用户启动 KeyDock
2. KeyDock 在屏幕底部显示一个置顶键盘面板
3. 用户点击一个虚拟键
4. KeyDock 保持不激活窗口
5. 当前前台窗口收到对应按键输入
6. 用户继续点击其他键或关闭 KeyDock

## 窗口行为

- 默认停靠在主显示器底部
- 保持置顶
- 点击键盘不抢焦点
- 不进入 Alt-Tab 主任务切换
- Esc 或关闭按钮可退出
- 后续版本再考虑拖动、吸附、多显示器记忆

## v1 必做

- 英文 QWERTY 全键盘
- 数字行
- 常用符号
- Backspace
- Enter
- Space
- Esc
- Shift
- Ctrl
- Alt
- 鼠标点击输入
- 触控点击输入
- 高 DPI 下布局清晰

## v1 不做

- IME 集成
- 中文候选词
- 拼音组合输入
- 宏录制
- 快捷命令面板
- 手势输入
- 滑动输入
- 皮肤市场
- 插件系统
- 系统服务
- 向管理员窗口绕过注入

## 默认布局

- 第一行：Esc、数字和常用符号
- 第二行：QWERTY 字母行
- 第三行：ASDF 字母行、Enter
- 第四行：Shift、ZXCV 字母行、Backspace
- 第五行：Ctrl、Alt、Space

布局优先稳定和可读，不追求完全复刻物理键盘。

## 视觉要求

- 按键有明确边界
- hover、pressed、active 三种状态必须可区分
- 修饰键锁定状态必须明显
- 字体使用系统 UI 风格
- 背景和按键对比清晰
- 高 DPI 下不糊、不挤、不截断文字

## 失败语义

- 如果系统拒绝输入注入，显示可理解错误，不 panic
- 如果目标窗口权限高于 KeyDock，说明这是 Windows 完整性级别限制
- 如果渲染设备丢失，重建渲染资源
- 如果 DPI 变化，重算布局

## 验收标准

- 打开 KeyDock 后能看到一个完整英文虚拟键盘
- 点击普通字母能输入到原前台窗口
- 点击 KeyDock 不会让目标窗口失焦
- Shift/Ctrl/Alt 状态能被用户看懂
- Notepad、Windows Terminal、浏览器输入框都能完成基础输入
