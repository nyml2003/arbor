# arbor-tui 性能水位

本文记录当前可执行的性能水位。

性能优化必须先补水位，再改实现。功能问题必须先补失败测试，再修实现。

## 当前水位

### 1. ANSI 编码水位

测试位置：

- `crates/arbor-tui-adapters/src/crossterm_backend.rs`

运行命令：

```bash
cargo test -p arbor-tui-adapters encode_waterline
```

当前水位：

| 场景 | dirty 输入 | dirty row | cursor move | style run | grid cell |
| --- | ---: | ---: | ---: | ---: | ---: |
| 单个 dirty cell，240 列同样式行 | 1 | 1 | 1 | 1 | 240 |
| 乱序/重复 dirty region，80 列两行 | 3 | 2 | 2 | 2 | 160 |

解释：

- backend 会把 dirty cell 扩成整行输出。
- 这是为了强制写入背景色，避免终端默认背景透出。
- 同一行同样式必须合并成一个 style run。
- 重复或乱序 dirty region 必须先收敛成唯一 dirty row。

不要为了减少输出列数而回退到局部 repaint。除非先用 xterm.js 等真实终端前端水位证明不会再漏终端底色。

### 2. 应用层帧水位

测试位置：

- `crates/arbor-tui-testing/tests/rendering_e2e.rs`

运行命令：

```bash
cargo test -p arbor-tui-testing --test rendering_e2e initial_render_reports_frame_stats_waterline
```

当前水位：

- 首帧渲染后 `FrameStats.frame_seq == 1`。
- 首帧渲染后 `FrameStats.dirty_regions > 0`。

解释：

- `FrameStats` 是应用层性能入口。
- 它记录 layout、render、diff、emit 和总耗时。
- 单元测试不使用微秒阈值。机器差异会让这种阈值不稳定。
- 真正要比较耗时，使用同一台机器、同一终端尺寸、同一场景做前后对比。

### 3. 屏幕热路径水位

测试位置：

- `crates/arbor-tui-domain/src/cell.rs`
- `crates/arbor-tui-domain/src/screen.rs`
- `crates/arbor-tui-domain/src/text.rs`

运行命令：

```bash
cargo test -p arbor-tui-domain cell_is_copy_value_type
cargo test -p arbor-tui-domain screen::tests
cargo test -p arbor-tui-domain char_width_measures_single_terminal_cell_width
```

当前水位：

- `Cell` 是 `Copy` 值类型。
- `fill_rect()` 必须正确裁剪到屏幕边界。
- `blit()` 必须正确裁剪到目标屏幕边界。
- `char_width()` 必须覆盖 ASCII、CJK 和 combining mark。

解释：

- `fill_rect()` 和 `blit()` 是渲染合成热路径。
- 测试锁定裁剪语义，允许内部实现改成行级 slice。
- `char_width()` 给单字符宽度计算提供无分配入口。

## 本期优化

本期只改热路径，不改变渲染语义：

- `VirtualScreen` 新增借用式读取 `cell_at_ref()`。
- diff 热路径用借用比较，避免每个 cell clone。
- crossterm backend 在 full-row expand 后不再二次 sort/merge。
- crossterm backend 复用 style run 的文本 buffer。
- `Cell` 和 `DirtyRegion` 改为 `Copy` 值类型。
- `fill_rect()`、`blit()`、`resize()` 改为按行 slice 填充/复制。
- crossterm backend 的 dirty row 扩展改为 bitmap 收敛。
- `char_width()` 替代单字符 `to_string()` 后再测宽。

## 后续优化规则

1. 先写水位。
2. 再改实现。
3. 保留 full-row repaint 语义。
4. 水位测试优先用计数，不用耗时阈值。
5. 耗时数据只作为人工对比，不作为默认 CI 硬阈值。
