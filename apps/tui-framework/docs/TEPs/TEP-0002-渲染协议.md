# TEP-0002: 渲染协议

## 本质

Virtual DOM diff 的终端等价物。React 在内存里建虚拟 DOM 树，diff 出变更，apply 最小 DOM 操作。我们在内存里建 VirtualScreen（二维字符网格），diff 出 DirtyRegion，emit 最小 ANSI 序列。

差异：React 的 diff 对象是树（节点可嵌套、重排）。终端 diff 对象是固定尺寸的字符网格（每个 Cell 位置确定，只有内容会变）。算法更简单——逐行比较即可，不需要 key、不需要 reconciliation。

## Cell — 渲染原子

终端屏幕上每个字符位置就是一个 Cell。六个属性：字符、前景色（256 色调色板 + 可选 TrueColor）、背景色、粗体/斜体/下划线等标记、以及一个"续列"标记（宽字符占两列时，第二列标记为续列，emit 时跳过——否则写空格会覆盖 CJK 右半部分）。

## VirtualScreen — 虚拟画布

二维 Cell 数组，行优先。所有组件渲染到这个画布上。渲染完成后，和上一帧的画布逐行比较 Cell 是否相等。相等的跳过，不等的输出 DirtyRegion（行号、起始列、结束列）。

## diff 策略

朴素逐行比较。终端 Cell 量级下（240×60 = 14,400）耗时 < 100μs，不需要 Myers 或 SIMD。

行列数不一致时：旧屏多出的行标记为整行清除，新屏多出的行标记为整行新建。

## 后端适配器

domain 层定义后端接口，infra 层实现。后端接收脏区列表和画布，排序合并后生成 ANSI 写入终端。末尾统一 flush。

两个实现：生产用 crossterm，测试用模拟后端（内存记录输出）。

## 闪屏对策

交替屏幕缓冲提供天然双缓冲。切换屏幕时用户看不到中间帧。光标默认隐藏，仅 Input 获焦点时显示。全屏刷新只通过 diff 局部更新，不主动清屏。

## 已决议

- Diff：朴素逐行，不做 SIMD
- 宽字符续列：Cell 标记，emit 跳过
- 终端颜色检测：v1 不做自动检测
- NO_COLOR：后端层拦截
