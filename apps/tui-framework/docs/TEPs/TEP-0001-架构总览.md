# TEP-0001: 架构总览

## 本质

一个终端字符网格上的保留模式 UI 框架。核心机制借用了四个成熟的 UI 模式，针对终端约束做了简化：

| 模式 | 来源 | 做了什么 | 砍了什么 |
|------|------|---------|---------|
| Virtual DOM + diff | React | 在 VirtualScreen 上逐行 diff，只 emit 变更区域 | 树 diff → 网格 diff（终端不需要 vdom 树） |
| 细粒度 Signal | SolidJS | 显式订阅 + DirtyTracker 标记脏组件 | 自动追踪 → 显式订阅（单线程不需要） |
| Flexbox 布局 | CSS | direction/justify/align/flex，标准盒模型 | wrap/gap/百分比（终端等宽网格不需要） |
| 事件冒泡 + 焦点 | DOM | Tab 序、焦点组件优先、冒泡到父级 | 捕获阶段、鼠标事件、focus/blur 事件 |

## 四条协议

**渲染协议（TEP-0002）**：VirtualScreen → diff → DirtyRegion → ANSI emit。Virtual DOM 的终端等价物。

**布局协议（TEP-0003）**：measure（子报约束）→ layout（父分空间）。Flexbox 的终端等价物。

**响应式协议（TEP-0004）**：Signal（写）→ ReadSignal（读）→ DirtyTracker（标记）→ 批渲染。手动版 SolidJS。

**组件协议（TEP-0005）**：生命周期（mount → 事件循环 → unmount）、焦点路由、事件冒泡。DOM 组件模型的终端等价物。

## 边界

这个框架为 Arbor 运维工具提供终端交付面。目标场景：运维人员在终端查看网络状态、系统配置。键盘操作，字符网格展示。

不做的事：
- 不做 mouse — SSH/tmux 下 mouse protocol 不转发
- 不做 IME — raw mode 不转发未完成组合序列
- 不做 TrueColor 依赖 — SSH/tmux 常降级到 8/16 色
- 不做复杂文字排版 — 只做 ASCII + 基础 CJK 列宽测量
- 不做 terminfo 穷举兼容 — 只支持现代 ANSI 终端

## 硬约束

启动 < 50ms。1080p 全屏帧 < 5ms。空闲 CPU 接近零。内存 < 5MB。二进制 ≤ 8MB 静态链接。退出时终端完全恢复。四个平台交叉编译。
