# TEP-0005: 组件协议

## 本质

DOM 组件模型的终端等价物。组件有生命周期（mount → 事件循环 → unmount），有焦点管理（tabindex + 焦点路由），有事件冒泡（Handled 停止，Bubble 沿父链向上）。

差异：终端只有键盘输入。没有鼠标事件。没有 CSS 伪类和选择器。组件集固定（8 种内置组件），不做自定义组件——枚举编译期穷举比 vtable 更合适。

## 组件树

固定组件集用枚举而非 trait object。每个 variant 是一个内置组件：Box（容器，只做布局）、Text（文本渲染）、Input（非受控输入框）、Button、List、Table、Tabs、ScrollView。

遍历由框架负责。组件不主动触及子或父——它只响应框架调用。

## 生命周期

和 React class component 一致：

- **mount**：插入树。订阅依赖的状态值。初始化内部状态。
- **事件循环**：渲染时提供布局和内容。按键时响应或忽略。
- **unmount**：移出树。退订状态。清理内部状态。
- **销毁**：兜底退订——unmount 未调用时也不泄漏。

## 焦点协议

和 HTML tabindex + focus 一致。可获焦点的组件按声明顺序排列。Tab 前进，Shift+Tab 后退。到达末尾循环回首。

单组件时 Tab 保持自身。焦点组件移除时自动清除焦点。焦点变更只标记新旧焦点组件脏——其他组件不重绘。

## 事件冒泡

和 DOM 事件冒泡一致。键盘事件首先发给焦点组件。组件处理：

- **已处理**：停止传播。标记自身脏。父级不收到事件。
- **未处理**：沿父链向上冒泡。每级父组件有机会处理。到根仍未被处理则丢弃。

全局快捷键（Tab、Shift+Tab、退出键）在冒泡前拦截——和 DOM 的 `event.preventDefault()` 之前系统级处理一致。

## 输入源

独立线程阻塞读终端输入。解析后通过固定容量通道推送主线程。主线程超时阻塞等待，零 CPU。连续重复输入合并（持续按方向键只保留最后一次），避免渲染抖动。

## 已决议

- 组件树用枚举，固定 8 个 variant
- 生命周期和 React class component 对齐
- 焦点循环，单组件保持自身
- 事件冒泡沿父链向上，Handled 停止
- 泛型 List/Table v1 不做——回调自定义渲染替代类型擦除
