# TEP-0004: 输入系统——键盘解析与焦点管理

## 摘要

单线程渲染，crossterm 负责 stdin 读取和 CSI 解析，通过 mpsc channel 推送 `KeyEvent` 到主线程。双 mpsc（事件 + 终止信号）。焦点系统支持事件冒泡、Tab/Shift+Tab 导航、组件销毁自动焦点转移。不拦截终端原生快捷键。

## 目标

用户在终端按下键盘 → CSI Parser 解析 → `KeyEvent` 入 channel → 主线程批量拉取 → 焦点分发 + 冒泡 → Widget `on_key` 回调 → signal 更新 → 触发重绘。

## 架构

```
独立线程: stdin (crossterm event::read) → KeyEvent → event_tx ──┐
           shutdown_rx ← 主线程终止信号                            │
                                                                   │
主线程:    event_rx ──→ 批量收集 → 合并节流 → 焦点分发 → signal.set() → render
           shutdown_tx (Drop 时发送)
```

独立线程只做一件事：阻塞读 stdin 字节，parse 成 `KeyEvent`，送进 channel。不接触组件树、不修改 signal、不做渲染。主线程在每次渲染循环迭代中非阻塞 `try_recv` 批量拉取。

## InputReader trait

```rust
// domain/input.rs

pub trait InputReader: Send {
    /// 非阻塞读取——返回积压的所有事件
    fn poll(&self) -> Vec<KeyEvent>;

    /// 阻塞等待下一个事件（独立线程使用）
    fn read_blocking(&self) -> KeyEvent;

    /// 发送终止信号，唤醒阻塞读线程并退出
    fn shutdown(&self);
}
```

infra 层提供两个实现：
- `StdinReader`（默认）：自建独立线程 + 逐字节 CSI 状态机，无 crossterm event loop 依赖
- `CrosstermStdinReader`（备选）：复用 crossterm 内置终端 IO，用于快速调试和兼容性验证
- `SimulatedInput`（测试）：事件回放队列

## KeyEvent 结构

```rust
// domain/input.rs

#[derive(Clone, PartialEq, Eq)]
pub struct KeyEvent {
    pub key: Key,
    pub modifiers: Modifiers,
}

#[derive(Clone, PartialEq, Eq)]
pub enum Key {
    Char(char),
    Enter,
    Tab,
    Backspace,
    Escape,
    ArrowUp,
    ArrowDown,
    ArrowLeft,
    ArrowRight,
    Home,
    End,
    PageUp,
    PageDown,
    Insert,
    Delete,
    F(u8),               // F1-F12
}

#[derive(Copy, Clone, PartialEq, Eq, Default)]
pub struct Modifiers {
    pub ctrl: bool,
    pub alt: bool,
    pub shift: bool,
}
```

`Key::Char` 不携带 modifier——组合键（`Ctrl+W`、`Alt+x`）走带 `Modifiers` 的独立 `Key` 变体（`Key::Char('w')` + `Modifiers { ctrl: true }`），不搞两套编码。

## Ctrl/Alt/Shift 按键映射

crossterm 已处理所有平台差异——框架只需将 `crossterm::event::KeyEvent` 映射到框架的 `KeyEvent`。映射函数见上方 `crossterm_to_keyevent()`。Modifiers 直接从 crossterm 的 `KeyModifiers` 提取，不自行解析字节序列。

## CSI 解析（crossterm）

CSI 解析和 stdin 读取全部由 crossterm 的 `event::read()` 处理。不自研状态机。crossterm 已封装跨平台兼容性——小众终端、SSH 会话、老旧 Linux tty 的序列差异都已被社区验证。

```rust
// infra/stdin_reader.rs
// 独立线程中调用 crossterm::event::read() 阻塞读取

use crossterm::event::{read, Event, KeyCode, KeyModifiers};

fn crossterm_to_keyevent(event: Event) -> Option<KeyEvent> {
    match event {
        Event::Key(key) => {
            let key = match key.code {
                KeyCode::Char(c) => Key::Char(c),
                KeyCode::Enter => Key::Enter,
                KeyCode::Tab => Key::Tab,
                KeyCode::Backspace => Key::Backspace,
                KeyCode::Esc => Key::Escape,
                KeyCode::Up => Key::ArrowUp,
                KeyCode::Down => Key::ArrowDown,
                KeyCode::Left => Key::ArrowLeft,
                KeyCode::Right => Key::ArrowRight,
                KeyCode::Home => Key::Home,
                KeyCode::End => Key::End,
                KeyCode::PageUp => Key::PageUp,
                KeyCode::PageDown => Key::PageDown,
                KeyCode::Insert => Key::Insert,
                KeyCode::Delete => Key::Delete,
                KeyCode::F(n) => Key::F(n),
                _ => return None,  // 过滤不支持的按键
            };
            let modifiers = Modifiers {
                ctrl: key.modifiers.contains(KeyModifiers::CONTROL),
                alt: key.modifiers.contains(KeyModifiers::ALT),
                shift: key.modifiers.contains(KeyModifiers::SHIFT),
            };
            Some(KeyEvent { key, modifiers })
        }
        Event::Resize(w, h) => {
            // resize 事件直接写入全局屏幕尺寸，不走 channel
            Some(KeyEvent::resize(w, h))
        }
        _ => None,  // mouse、focus 等事件直接丢弃
    }
}
```

crossterm 事件映射到框架的 `KeyEvent`——过滤掉不支持的按键类型（mouse、focus）。`Event::Resize` 不走 channel，直接写入全局屏幕尺寸触发 SIGWINCH 处理。

## 事件合并节流

按键重复（hold 方向键）每秒 ~30 个相同事件。合并规则标准化：

### 合并（仅保留最后一个）

- 连续的相同方向键（`ArrowUp/Down/Left/Right`、`PageUp/PageDown`）
- 连续的相同 `Char(c)`（可打印字符）

### 不合并（逐个下发）

- `Backspace`、`Enter`、`Escape`、`Tab`
- 所有 `F1`-`F12`
- 任意带 modifier（Ctrl/Alt/Shift）的按键——全部独立事件

### 合并链规则

间隔插入其他类型按键则断开合并链。`a a Up a` → `[a, Up, a]` 三个独立事件（第二个 `a` 不与第一个 `a` 合并，因为中间插入了 `Up`）。

## 焦点系统

### Widget trait 聚焦接口（同步对齐 TEP-0003）

```rust
// domain/widget.rs

pub trait Widget {
    // ... 布局相关方法见 TEP-0003

    fn focusable(&self) -> bool { false }
    fn tab_index(&self) -> u16 { 0 }

    /// 键盘事件回调。返回 Handled 消费事件，Bubble 向上冒泡
    fn on_key(&mut self, event: &KeyEvent) -> KeyHandleResult { KeyHandleResult::Bubble }
}

pub enum KeyHandleResult {
    Handled,   // 事件已消费，停止冒泡
    Bubble,    // 继续向上冒泡到父级
}
```

### FocusManager

```rust
// domain/focus.rs

pub struct FocusManager {
    focus_path: Vec<usize>,         // 从根到当前焦点 widget 的索引路径
    tab_order: Vec<WidgetId>,       // 所有 focusable widget 按 tab_index + DFS 排序
}

impl FocusManager {
    pub fn next(&mut self) { /* Tab 前进 */ }
    pub fn prev(&mut self) { /* Shift+Tab 后退 */ }
    pub fn focus(&mut self, id: WidgetId) { /* 跳转，id 不在 tab_order 则忽略 */ }
    pub fn blur(&mut self) { /* 取消当前焦点 */ }
    pub fn current(&self) -> Option<WidgetId> { /* 当前焦点 widget ID */ }

    /// 重新遍历组件树生成 tab 导航序列
    pub fn rebuild_tab_order(&mut self, root: &dyn Widget);
}
```

### Tab 序规则

1. 收集所有 `focusable() == true` 的 Widget
2. 按 `tab_index` 升序排列；同 `tab_index` 按 DFS 深度优先遍历顺序
3. `rebuild_tab_order()` 触发条件：组件树结构变更（增删节点）、任意 Widget 的 `focusable`/`tab_index` 属性修改。窗口 resize、布局重算不触发重建

### 焦点导航边界规则

- 无可聚焦组件：Tab/Shift+Tab 空操作，无视觉反馈
- 仅单个可聚焦组件：Tab/Shift+Tab 保持当前焦点不切换
- 组件销毁时若为当前焦点：自动 blur，跳转至 tab_order 中下一个有效节点
- 调用 `focus(id)` 若 id 不在 tab_order 列表：静默忽略

### 事件分发与冒泡

优先级：

```
1. 当前焦点 Widget → on_key(event)
     ├── Handled → 停止
     └── Bubble  → 继续
2. 父级 Widget → on_key(event)
     ├── Handled → 停止
     └── Bubble  → 继续
3. ... 递归到根

无焦点时：Tab/Shift+Tab 仍由 FocusManager 处理（初始化选中第一个可聚焦组件）；其余按键丢弃

系统信号（SIGINT/SIGTSTP）：不进入焦点分发，由 app 层信号回调单独处理
```

## 信号处理

统一信号管理器 `SignalManager`（app 层集中管理所有 OS 信号回调）：

```rust
// app/signal_manager.rs

pub struct SignalManager {
    pub on_quit: Option<Box<dyn FnOnce()>>,   // SIGINT 退出前回调
    pub on_resize: Option<Box<dyn Fn(u16, u16)>>,
}

impl SignalManager {
    pub fn install() -> Self { /* 注册 SIGINT/SIGTSTP/SIGWINCH handler */ }
}
```

| 信号 | 处理 |
|------|------|
| **SIGINT** (Ctrl+C) | `SignalManager` 拦截。若设 `on_quit` 回调则调用，否则直接退出。`TerminalGuard::Drop` 保证终端恢复 |
| **SIGTSTP** (Ctrl+Z) | Drop `TerminalGuard` 恢复终端 → `SIGSTOP` 自身。`SIGCONT` 后：重新创建 `TerminalGuard` → 进入 raw mode + alternate screen → **标记所有 widget 全脏**（`app.mark_all_dirty()`）→ 全屏重绘。不依赖旧 VirtualScreen 缓存 diff——全脏标记确保重绘完整 |
| **SIGWINCH** | `SignalManager` 回调更新屏幕 Size → 全树 re-measure + re-layout → `mark_all_dirty()` → 全屏重绘。tab_order 不重建 |

SIGTSTP 恢复后强制全脏标记是必须步骤——不可省略。否则旧 VirtualScreen 与新终端状态不匹配，diff 误判无变更导致界面空白。

## 终端快捷键边界

框架不拦截以下组合键——由终端/shell/OS 处理：

- `Ctrl+C` → SIGINT（框架设 quit 回调，不"劫持"）
- `Ctrl+Z` → SIGTSTP
- `Ctrl+D` → EOF（shell 层）
- `Ctrl+L` → 终端清屏（部分终端）
- `Ctrl+S / Ctrl+Q` → 软件流控（XON/XOFF）

框架层快捷键在组件树内生效（`j/k` 移动列表选择等），不跟终端冲突。Ctrl+W 删词逻辑：框架不内置，仅原样下发 `KeyEvent { key: Char('w'), modifiers: { ctrl: true } }`，由 Input 等组件自行实现单词边界逻辑。

## CPU 闲置节流

主线程使用阻塞等待而非空循环。Stdin 独立线程在有输入时通过 mpsc 推送事件。主线程用 `recv_timeout(100ms)`：

```
loop {
    match event_rx.recv_timeout(Duration::from_millis(100)) {
        Ok(events) => { /* 批量处理 */ },
        Err(Timeout) => { /* 检查定时器，无则继续等待 */ },
    }
    runtime.render_if_dirty(&mut backend);
}
```

100ms 超时保证：无输入时 CPU 使用率接近零；有输入时响应延迟 ≤ 100ms（人无法感知）；定时器（如 netmon 每秒刷新）通过超时唤醒后的检查触发。

## 事件缓冲区上限

mpsc channel 使用 bounded channel，容量 256 个事件：

- 正常情况：channel 从不堆积。人类输入速率远低于处理速率
- 极端情况（二进制数据误写入 stdin）：channel 满时 `send()` 返回错误，stdin 线程丢弃最旧事件（清空缓冲区后重新积累）。不会因缓冲区无界导致内存泄漏
- `merge_events` 在 channel 积压后自动合并连续重复事件，进一步压缩

## 与渲染循环的集成

```
主循环（app/event_loop.rs）：

loop {
    // 1. 阻塞等待，超时 100ms
    let events = match event_rx.recv_timeout(Duration::from_millis(100)) {
        Ok(batch) => batch,
        Err(Timeout) => {
            check_timers();  // 定时器回调（如 netmon 1s 刷新）
            runtime.render_if_dirty(&mut backend);
            continue;
        }
    };

    // 2. 合并 + 节流
    let merged = merge_events(&events);

    // 3. 逐个分发：焦点 Widget 冒泡链
    for event in merged {
        focus_manager.dispatch(&event, &widget_tree);
    }

    // 4. signal 脏区收集 → layout → diff → emit
    runtime.render_if_dirty(&mut backend);
}
```

## 附录 A：线程通道生命周期

```rust
// infra/stdin_reader.rs

pub struct StdinReader {
    event_tx: Sender<KeyEvent>,
    shutdown_tx: Mutex<Option<Sender<()>>>,
    handle: Option<JoinHandle<()>>,
}

impl StdinReader {
    pub fn new() -> Self {
        let (event_tx, event_rx) = mpsc::channel();
        let (shutdown_tx, shutdown_rx) = mpsc::channel::<()>();

        let handle = thread::spawn(move || {
            loop {
                // 同时等待 stdin 输入或关闭信号
                // Unix: select/poll on stdin fd + shutdown_rx
                // Windows: WaitForMultipleObjects
                if shutdown_rx.try_recv().is_ok() {
                    break;
                }
                if let Some(event) = read_stdin_byte() {
                    event_tx.send(event).ok();
                }
            }
        });

        Self { event_tx, shutdown_tx: Mutex::new(Some(shutdown_tx)), handle: Some(handle) }
    }
}

impl Drop for StdinReader {
    fn drop(&mut self) {
        self.shutdown();  // 发送终止信号
        if let Some(h) = self.handle.take() {
            h.join().ok();  // 等待线程退出
        }
    }
}
```
