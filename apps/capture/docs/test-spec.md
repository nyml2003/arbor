# Capture 测试规格

这份文档定义实现后的验收场景。当前阶段只写规格，不跑测试。

## 核心场景

### 1. 区域截图成功

1. 用户按全局快捷键
2. overlay 出现
3. 用户框选一个区域
4. 应用完成截图
5. 剪贴板里出现图片
6. 系统通知出现
7. 点击通知后，系统默认图片查看器打开本次缓存图

通过标准：

- 剪贴板内容是图片
- 缓存文件存在
- 通知点击目标正确

### 2. 当前屏幕截图成功

1. 用户从托盘菜单触发当前屏幕截图
2. 应用直接完成截图
3. 后续行为与区域截图一致

通过标准：

- 不经过框选
- 复制、缓存、通知三步都成功

## 失败和取消

### 3. 用户取消截图

1. 用户触发区域截图
2. overlay 出现
3. 用户按 `Esc` 或取消操作

通过标准：

- overlay 关闭
- 不修改剪贴板
- 不写缓存文件
- 不发通知

### 4. 快捷键注册失败

前提：

- 默认快捷键被别的程序占用

通过标准：

- 应用能明确提示注册失败
- 设置页仍可修改快捷键
- 应用不进入静默失效状态

### 5. 剪贴板写入失败

通过标准：

- 截图文件仍然可以生成
- 错误以结构化方式暴露
- 不因剪贴板失败导致应用崩溃

### 6. 通知发送失败

通过标准：

- 截图和剪贴板流程仍然完成
- 失败被记录为普通失败路径
- 不影响下一次截图

## 一致性场景

### 7. 连续截图

1. 用户快速连续触发多次截图

通过标准：

- 每次截图都有独立缓存文件
- 最近一次截图记录正确
- 通知点击不会打开错图

### 8. 缓存清理

前提：

- 缓存数量超过设置上限

通过标准：

- 旧文件被清理
- 最新截图保留
- 最近一次截图路径仍可用

## 平台范围

- Windows：v1 正式验收平台
- macOS/Linux：当前只做架构预留，不作为本阶段通过条件
