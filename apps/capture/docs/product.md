# Capture 产品说明

## 一句话

Capture 是一个常驻桌面截图工具。它优先解决“快截一下就走”的场景，不把首版做成编辑器或资料库。

## 目标

做一个轻量、稳定、默认无主窗口的截图工具。当前 MVP 只支持 Windows，不保留跨端兼容目标。

## 目标用户

当前目标用户就是自己。重点场景是：

- 写文档时快速截一张图
- 和人沟通时快速贴一张图
- 临时留下一张图，再用系统图片查看器看原图

## v1 主流程

### 区域截图

1. 用户按全局快捷键
2. 应用拉起透明 overlay
3. 用户拖拽一个矩形区域
4. 用户松开鼠标，确认截图
5. 应用把截图复制到剪贴板
6. 应用发系统通知
7. 用户点击通知时，系统默认图片查看器打开这张缓存图

### 当前屏幕截图

1. 用户从托盘菜单触发“当前屏幕截图”
2. 应用直接抓取当前屏幕
3. 后续行为与区域截图一致

## 默认行为

- 默认结果是“复制到剪贴板”
- 同时写一份缓存文件，供通知点击后打开
- 默认不弹保存框
- 默认不进入应用内历史页

## 交互边界

- 默认无主窗口，常驻托盘
- overlay 只在截图时出现
- 设置页按需打开
- 点击通知后直接打开系统图片查看器，不跳回应用

## 前端策略

### Overlay

- 用本地静态页面
- 不做 hydration
- 不挂 Solid runtime
- 只绑定鼠标和键盘事件

这部分要尽量薄。目标不是做一个“前端页面”，而是做一个能最快拉起的框选层。

### Settings

- 单独开窗口
- 按需加载
- 可以使用 SolidJS

设置页不参与截图首屏，不应该影响截图拉起速度。

## v1 必做

- 全局快捷键
- 区域截图
- 当前屏幕截图
- 复制到剪贴板
- 系统通知
- 点击通知后打开缓存图
- 托盘菜单
- 设置页

## UI 性能要求

- overlay 的 DOM 层级保持很浅
- overlay 的 CSS 只做必要的遮罩、选区框和坐标反馈
- 避免重阴影、模糊、大量嵌套容器
- 避免让设置页代码进入 overlay 首屏

## v1 不做

- 标注
- OCR
- 录屏
- 历史资料库
- 标签和搜索
- 云同步
- CLI

## 设置项

- 截图快捷键
- 是否发送通知
- 缓存保留上限
- 是否开机启动

## 缓存策略

- 每次截图都写入缓存目录
- 文件名使用时间戳
- 默认保留最近 50 张
- 超出上限后自动清理旧文件

## 验收标准

- 用户触发截图后，可以在 1 次交互里完成截图并粘贴
- 截图完成后，剪贴板里是图片，不是文件路径
- 点击通知能稳定打开本次截图文件
- 取消截图不会污染剪贴板，也不会留下多余缓存文件
