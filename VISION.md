# Vision — Arbor 终态全景

## 引擎模型

Arbor 不是一堆工具，是一台**持续运转的个人产出引擎**。四个环节构成闭环：

```
build
  │  做出来
  ▼
learn
  │  学到了什么、形成了什么模式
  ▼
manage
  │  下一步做什么、优先级是什么
  ▼
show
  │  把结果展示给外界、获得反馈和动力
  ▼
build
  ...
```

## 每个环节的定义和边界

### build
- **职责**：把想法变成可用的东西。代码、工具、原型、实验。
- **边界**：只管「做」，不管「记」和「管」。
- **终态**：有稳定的工具链、框架、构建流程；新工具能快速落地。

### learn
- **职责**：捕获经验、形成模式、记录知识。笔记、架构决策、最佳实践、复盘。
- **边界**：只管「记」，不管「做」和「推」。
- **终态**：有一套结构化的知识库；做任何东西都有文档沉淀；经验可复用、可检索。

### manage
- **职责**：组织一切。任务列表、路线图、优先级、资源索引。
- **边界**：只管「排」，不管「做」和「记」。
- **终态**：打开 Arbor 就知道下一步做什么；每个东西在树上都有位置；不迷路、不遗忘。

### show
- **职责**：向外界展示。可浏览的作品集、可分享的文档、可下载的工具。
- **边界**：只管「展」，不管「做」的内容本身。
- **终态**：别人打开链接就能浏览你的工具和作品；树结构本身就是导航。

## 终态描述

打开 Arbor：

```
arbor/
├── build/             ← 你做的工具和项目
│   ├── tool-a/        ←   描述、源码、二进制
│   ├── tool-b/
│   └── ...
├── learn/             ← 你积累的经验
│   ├── patterns/      ←   架构模式、设计原则
│   ├── notes/         ←   学习记录、技术调研
│   └── retrospectives/←   项目回顾、经验总结
├── manage/            ← 你的计划
│   ├── tasks/         ←   当前在做的事
│   ├── roadmap/       ←   长期计划
│   └── resources/     ←   工具索引、配置
└── show/              ← 对外窗口
    ├── portfolio/     ←   成品 demo、截图
    ├── articles/      ←   写作输出
    └── about/         ←   个人介绍
```

四个环节互相推动。做完一个工具（build）→ 记下经验（learn）→ 更新计划（manage）→ 发布到作品集（show）→ 反馈驱动下一轮（build）。

## 驱动问题

每次做一件事之前，问自己：

> 做了这件事，四个环节里哪个会更强？

如果回答不上来，说明这件事可能不值得做。
