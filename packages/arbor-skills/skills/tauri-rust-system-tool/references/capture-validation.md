# 采集验收

## 意图

用用户流程和失败路径验证系统工具，而不是只检查能启动。

## 适用场景

- 实现截图、剪贴板、通知、缓存。
- 修改快捷键、托盘、overlay、settings。
- 收尾验收 capture 改动。

## 必须遵守的规则

- 区域截图成功后，剪贴板里是图片，缓存文件存在，通知点击目标正确。
- 当前屏幕截图不经过框选，但复制、缓存、通知三步都要成功。
- 用户取消截图不能污染剪贴板，不能留下缓存文件，不能发通知。
- 快捷键注册失败要明确提示，不能静默失效。
- 连续截图要确保最近一次截图记录正确。

## 推荐模式

- 分别验证成功、取消、快捷键失败、剪贴板失败、通知失败、缓存清理。
- 缓存超过上限后清旧文件，但保留最新截图。
- Windows 作为 v1 正式验收平台。

## 反模式

- 只验证窗口能打开。
- 把通知失败当成截图失败。
- 连续截图时通知点击打开错图。
