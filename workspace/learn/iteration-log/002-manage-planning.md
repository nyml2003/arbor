# 002 - Manage 任务规划

日期：2026-06-07

## 做了什么

用 agent 能力在 `manage/` 下建立了 Arbor 的任务清单（`manage/tasks.md`），重新评估了四个环节的完成度。

## 修正

用户纠正了两个关键认知：

1. **「不要归档，直接删」** — 大多数旧项目是学习产物，代码没有保留价值。不需要归档目录，直接 `Remove-Item`。
2. **「经验还没内化呢」** — 经验在代码里，还没抽出来。不能说「经验已内化」。正确的顺序是：**agent 先读代码、抽经验、写入 learn/，然后用户删项目。**

## 决策

1. **manage 也用 agent 能力** — 任务清单是纯 Markdown 文件，agent 负责整理、排期、更新。不需要 `@arbor/manage-core` 代码包。
2. **删文件必须用户执行** — agent 不删文件。任务清单里给出了完整的 PowerShell 删除命令，用户自己跑。
3. **从经验沉淀开始** — 最有价值的 8 个项目（ObolosFS、workshop、work-context 等）需要先抽经验再删。agent 负责读代码写模式文档。

## 下一步

按任务清单的第一优先级：agent 逐一读取旧项目代码，抽取模式，写入 `learn/patterns/`。
