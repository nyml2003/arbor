# 010 - 接上 MVP 三件套

日期：2026-06-07

## 做了什么

1. 接入 `arboard`，把生成的截图文件再写进系统剪贴板
2. 接入全局快捷键插件，让默认快捷键直接拉起 overlay
3. 新增右下角 toast 窗口，模拟系统通知入口
4. toast 点击后直接打开图片文件
5. 当前屏幕截图托盘项改成真实执行，而不是回退到 settings

## 学到了什么

### 对这个工具来说，“能粘贴”比“能保存”更像 MVP

写文件只是中间结果。真正决定工具能不能开始用的，是截完以后能不能立刻粘贴。这一步接上以后，工具才开始接近真实使用场景。

### 自定义 toast 比系统通知更适合现在这个阶段

你要的关键交互不是“桌面消息长什么样”，而是“点一下能重新看到图”。  
自定义 toast 窗口把这个交互完全掌握在应用自己手里，落地速度更快，也更容易调试。

### 快捷键、截图、剪贴板必须一起验证

这三块单独看都不复杂，但用户体验是串起来的。只做其中一块，工具仍然不可用。MVP 阶段应该优先把整条链串通，而不是把单点做得很精致。

## 决策

1. 当前通知实现先用自定义 toast，不阻塞 MVP
2. 先把“截完可粘贴”做成，再回头打磨通知外观
3. 全局快捷键默认读取设置值，设置变更时重新注册
