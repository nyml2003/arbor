# 008 - 打通 overlay 到 Rust 的最小选区回路

日期：2026-06-07

## 做了什么

1. 让 overlay 在松开鼠标时把选区矩形发给 Rust
2. Rust 把最后一次选区存进应用状态
3. 提交选区后自动关闭 overlay
4. settings 页增加“最近一次选区”展示和刷新按钮
5. settings 页在获得焦点时自动刷新最近选区

## 学到了什么

### 先打通坐标回路，比先接真实截图更值

这一步没有碰抓屏、图像裁剪或剪贴板。目标只有一个：验证这条链真的通了。

```text
overlay -> Tauri command -> Rust state -> settings 读取
```

这条链先通，后面接真实截图时就不会混淆“是系统截图没通”还是“前后端边界没通”。

### settings 是最便宜的可视化验证面

overlay 本身只负责框选，不适合塞太多调试信息。  
settings 本来就是低频页面，用它展示最近一次选区，既便宜，也不会污染 overlay 的首屏。

### 空壳阶段也应该保留真实的收口动作

提交选区后立刻关闭 overlay，这个行为虽然简单，但很重要。  
如果连这种最基本的收口动作都不做，后面真实截图接进来时还要再返工一次窗口流程。

## 决策

1. 最近一次选区放进 Rust 状态，而不是放前端全局变量
2. overlay 提交后立刻关闭，不停留在屏幕上
3. settings 负责展示调试态，overlay 继续保持极简
