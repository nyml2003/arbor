# 009 - 打通选区到 PNG 文件

日期：2026-06-07

## 做了什么

1. 给 Rust 侧接入 `xcap`
2. 给 Rust 侧接入 PNG 文件生成
3. 让 `submit_area_selection` 在存下选区后直接生成截图文件
4. 把最近一次截图结果放进应用状态
5. 让 settings 页展示最近一次截图文件路径和尺寸
6. 让“打开最近截图”调用系统默认图片查看器

## 学到了什么

### 先落文件，比先接剪贴板更容易验证

截图落成 PNG 文件以后，验证链路很直接：

```text
框选 -> Rust 抓屏 -> Rust 写文件 -> settings 显示路径 -> 系统查看器打开
```

这条链一旦通了，后面的剪贴板和通知只是再往后接两个副作用，不会反过来影响截图服务本身。

### settings 页适合做“结果面板”

overlay 负责快进快出，不适合显示太多结果信息。  
settings 页正好可以承担“最近一次选区”和“最近一次文件结果”的可视化验证，不会污染首屏。

### 先用主显示器区域抓屏，后面再修多显示器

当前版本先按主显示器打通区域截图。这不是终态，但足够验证截图服务、文件输出和打开文件这三层已经接上。多显示器对齐留到后面专门修。

## 决策

1. 当前阶段优先打通“选区到文件”，不同时接剪贴板和通知
2. 打开最近截图先直接调用系统默认图片查看器
3. 多显示器精确对齐暂时后放，不阻塞主链路验证
