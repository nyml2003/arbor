# 007 - Capture 空壳初始化

日期：2026-06-07

## 做了什么

1. 给 `apps/capture/` 补了真正的 package 和构建脚本
2. 建了 Vite 多入口：`overlay.html` 和 `settings.html`
3. 把 overlay 定成静态页面 + 原生事件
4. 把 settings 定成独立的 Solid 页面，并且按需加载
5. 建了 Tauri 配置、Cargo 清单、capability 文件和最小 Rust 壳
6. 接通了托盘、窗口显示隐藏和设置读写的空实现

## 学到了什么

### “空壳”也应该是真的壳

如果只建 README 和空目录，后面实现时还要重新决定：

- 包怎么起
- 窗口怎么配
- 托盘怎么挂
- 多入口怎么拆
- 前端和 Rust 的边界怎么接

这些问题不属于截图逻辑，但会反复打断真正的实现。先把壳搭起来，后面可以只盯主链路。

### overlay 和 settings 不能共用一个前端心智模型

overlay 追求的是首屏快、DOM 浅、CSS 简单、JS 少。  
settings 追求的是表单可维护。它们都属于“前端”，但不是一类页面。把它们强行揉成同一套入口，只会让 overlay 多付一层不必要的代价。

### Tauri 空壳已经足够承载下一阶段

现在已经有这些东西：

- 隐藏的 settings 窗口
- 隐藏的 overlay 窗口
- 托盘菜单
- Rust 命令入口
- 前端入口拆分

下一步只需要把真实的截图服务接进来，不需要再回头重搭宿主。

## 决策

1. overlay 保持无框架静态入口
2. settings 单独使用 Solid
3. 先让托盘和窗口能动，再接截图主链路
4. 当前 tray 中未实现的动作先回退到打开 settings，不假装功能已经完成
