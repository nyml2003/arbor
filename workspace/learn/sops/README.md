# SOPs — Arbor 四环节操作流程

每个环节有一条 SOP，描述该环节的标准操作流程。

## 预期的 SOP

| 环节 | SOP | 状态 |
|------|-----|------|
| build | 从头做一个新工具的流程：需求澄清 → 架构设计 → 实现 → 沉淀 | 待写（需要先走通一次完整流程） |
| learn | 从一个项目抽取经验的流程：选项目 → 读代码 → 识别模式 → 写 pattern → 审核 | ✅ `learn-extraction.md` |
| manage | 做一次任务规划的流程：回顾现状 → 列出选项 → 排优先级 → 更新任务清单 | 待写 |
| show | 把内容透出的流程：选要展示的内容 → 生成静态页面 → 验证 → 发布 | 待写 |

## 为什么现在都是"待写"

SOP 是实践后总结的——先走通，再写流程。不走通就写 SOP 是倒置的。
